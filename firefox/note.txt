IMPORTANT:

In the array containing the content scripts in the manifest, the following
order must be preserved if new files are created.

["settings/settings.js",
 "scripts/textScanningModel.js",
 <ALL TEXT SCANNER FILES>,
 "scripts/module.js",
 <ALL MODULE FILES>,
 "scripts/moduleFactory.js",
 "scripts/main.js"]

How to create a new module:

1. Create your module in a new file. It must extend the "Module" class.

2. Create a constructor which defines an attribute of the class called
   "options".

   The "options" object's keys must match those of the module-specific
   settings you wish to define.

   The "options" object's values are objects themselves. The keys of which are
   options as you want them to appear on the settings page, and the values of
   which are the variable values that will be assigned to the corresponding
   entry in the settings object. The first entry in the object should always
   be "type", which can currently have one of three acceptable values:
   "select_exclusive" : Causes options to appear on radio buttons in the page.
   "select_inclusive" : Causes options to appear on check boxes in the page.
   "text" : Accepts input from a text box.
   You must also specify a default value and define a moduleName key in the
   options.

   Ex.
   this.options = {
       highlightText : {
           moduleName : "Example Module",
           "type": "select_exclusive",
           "default" : true,
           "Highlight suspected text" : true,
           "Don't highlight suspected text" : false
       }
   }

3. The module must implement the scanPage() method to operate on a web page.
   The details of this implementation are left to the programmer.

4. In the ModuleFactory class, increase maxModules by 1.

5. In the ModuleFactory class, add a case in the switch statements insude
   the functions for your module. The contents should look the same as the
   contents of the other cases, except with your module name.

6. In the file "settings_page.html", add a <script> element that references
   your module's definition. Make sure to add it after the <script> element
   for "module.js" and before the <script> element for "moduleFactory.js".

7. In the manifest file, add a reference to your module's definition file
   to the content scripts array. Remember to add it after "module.js" and
   before "moduleFactory.js".
